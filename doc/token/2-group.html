模型与令牌

令牌分组介绍

「令牌分组」是寰星云科 API 控制访问范围的主要机制。每条令牌都属于一个分组，每个分组对应一组可用模型。换句话说：令牌分组 = 这条令牌能用的模型集合。

当前可用分组

分组名包含的模型典型用途
default大多数对话模型，包括 Claude / GPT / Gemini 主流型号日常开发、个人 Claude Code、Codex、Gemini CLI

当前文档中的令牌创建步骤统一选择 default 分组。实际可用模型范围以控制台「分组」页签显示为准。

新建令牌时挑分组

创建令牌时，分组字段决定接下来这条令牌能调到哪个模型集合。当前只需要选择 default 分组：

  • Claude Code、Codex、Gemini CLI、Cline 等客户端都使用 default 分组令牌。
  • 多人共用时，建议给团队成员各自发独立令牌，但都放在 default 分组里，方便审计。
  • 如果模型调用失败，先确认该模型是否已在 default 分组开放。

常见错误信息

model_not_found / model is not allowed in this group：当前令牌所属分组里没有这个模型。请确认令牌属于 default 分组，并联系管理员把模型加入该分组。

group_disabled：分组被临时关停。常见于上游故障应急或商务调整，等管理员重新开启即可。

分组与计费

模型倍率以控制台模型广场显示为准。当前文档统一使用 default 分组，因此创建令牌和查看倍率时都选择 default。

怎么知道某个客户端必须用哪个分组？

  1. 找到该客户端的目标模型名（如 Claude Code 默认使用 Claude Sonnet 系列）。
  2. 到模型广场搜索该模型，确认「所在分组」列里包含 default。
  3. 创建或使用 default 分组令牌调用该模型。
