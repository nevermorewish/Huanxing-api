创建 API 令牌

令牌（Token / API Key）是客户端调用 API 的身份凭证。建议为每一个客户端、每一个用途单独创建一个令牌，互相隔离额度、模型范围与有效期。

新建令牌

  1. 进入控制台「令牌 / API Keys」页面，点击「新建令牌」。
  2. 填写名称：建议带上用途与机器名，便于在日志和账单里区分。
  3. 选择分组：统一选择 default，详见 令牌分组介绍。
  4. 设置额度：可以选「不限额」，也可以填写一个上限（如 100 元），令牌跑满上限后自动失效。
  5. 设置过期时间：默认永久；建议为机器、CI、共享调试令牌设一个较短的过期时间。
  6. 点击「创建」，弹窗会显示 sk-... 格式的令牌字符串。

只展示一次：令牌完整字符串只会在创建成功时弹窗显示一次。请立刻复制保存到密码管理器或 1Password。关闭弹窗后再也看不到完整内容，只能重新生成。

令牌的形态

寰星云科 API 的令牌通常以 sk- 开头，整段长度约 48~64 位，例如：

sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

这段字符串就是 OpenAI 兼容 API 中的 Authorization: Bearer sk-...，也是 Claude Code 的 ANTHROPIC_AUTH_TOKEN 与 Gemini CLI 的 GEMINI_API_KEY 实际填的内容。

令牌泄露怎么办

如果不小心把令牌粘贴到 GitHub、聊天群或日志里，请立即回到「令牌」列表，把这条令牌「禁用」或「删除」，然后新建一条替换它，并把新令牌重新配置到所有客户端。
