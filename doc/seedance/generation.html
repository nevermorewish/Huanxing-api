视频模型

doubao-seedance-2.0 视频生成

doubao-seedance-2.0 是异步视频生成模型，提交请求后返回任务 ID，再通过任务状态接口查询进度与结果。它支持文生视频、图生视频（首帧 / 尾帧）、参考视频、参考音频与有声视频，并可输出横屏、竖屏、方形、超宽屏等多种比例。

  • 异步处理模式，返回任务 ID 用于后续查询。
  • 支持文生视频、图生视频（首帧 / 尾帧）。
  • 支持参考视频、参考音频、有声视频。
  • 支持横屏、竖屏、方形、超宽屏、自适应等多种比例。

认证

所有接口均需使用 Bearer Token 认证。在请求头中加入：

Authorization: Bearer sk-你的令牌

请求参数

参数类型说明
modelstring，必填视频生成模型名称，见下方「支持的模型」。
promptstring视频内容描述。文生视频时必填，图生视频 / 视频参考生视频时可选。建议明确主体、动作、镜头和风格。限制 4000 字符，建议 500 字符以内。
durationinteger视频时长（秒），范围 4 ~ 15，默认 5。
sizestring视频宽高比，默认 16:9。见下方「比例」。
resolutionstring视频分辨率，默认 480p。可选 480p / 720p / 1080p（1080p 仅 doubao-seedance-2.0-face 与 doubao-seedance-2.0 支持）。
seedinteger随机种子。相同请求下相同 seed 生成类似结果（不保证完全一致），不同 seed 生成不同结果。
generate_audioboolean是否生成有声视频，默认 false。为 true 时视频包含 AI 生成的配套音频。
return_last_frameboolean是否返回尾帧图片，默认 false。为 true 时结果额外返回最后一帧图片 URL，可用于连续视频生成。
toolsarray<object>工具列表，用于联网搜索等增强能力。元素含 type 字段，可选 web_search（生成时参考网络信息）。示例 [{"type": "web_search"}]。
image_urlsarray<string>图片 URL 数组，用于图生视频。最多 9 张。与 image_with_roles 不能同时使用。
image_with_rolesarray带角色的图片数组，支持指定首帧 / 尾帧 / 参考人像。与 image_urls 不能同时使用。
video_urlsarray<string>参考视频 URL 数组。最多 3 个，1.8s < 总时长 < 15.2s，分辨率 480P ~ 720P，不可出现真人。使用首 / 尾帧图片时不可用。
audio_urlsarray<string>参考音频 URL 数组。最多 3 个，总时长 ≤ 15s，需与参考图片或参考视频配合使用。使用首 / 尾帧图片时不可用。

支持的模型

model说明
doubao-seedance-2.0标准版，支持文生视频、图生视频、首尾帧视频、参考视频、参考音频、有声视频。
doubao-seedance-2.0-fast快速版，功能与标准版一致，生成速度更快。
doubao-seedance-2.0-face支持上传真人版，功能与标准版一致。
doubao-seedance-2.0-fast-face支持上传真人版，功能与快速版一致。

比例（size）

size用途
16:9横屏
9:16竖屏
1:1方形
4:3传统比例
3:4竖向传统比例
21:9超宽屏
adaptive自适应（根据输入图片 / 视频自动匹配）

image_with_roles 字段

数组元素包含两个字段：

  • url（string，必填）：图片 URL。支持普通图片 URL，或审核通过的 Asset URL（asset://asset_a，仅 doubao-seedance-2.0 与 doubao-seedance-2.0-fast 支持）。
  • role（string，必填）：图片角色，可选 first_frame（首帧）、last_frame（尾帧）、reference_image（参考人像，配合 Asset URL 使用）。
[
  { "url": "https://example.com/day.jpg", "role": "first_frame" },
  { "url": "https://example.com/night.jpg", "role": "last_frame" }
]

Asset URL（asset://...）来自审核通过的人像素材，仅 doubao-seedance-2.0 与 doubao-seedance-2.0-fast 模型可用。使用首 / 尾帧图片（image_with_roles）时，参考视频与参考音频不可用。

响应

字段类型说明
codeinteger响应状态码，成功时为 200。
dataarray返回数据数组。
data[].statusstring任务状态，初始提交时为 submitted。
data[].task_idstring任务唯一标识符，用于查询任务状态和结果。

使用场景

场景 1：文生视频

{
  "model": "doubao-seedance-2.0",
  "prompt": "小猫对着镜头打哈欠",
  "resolution": "720p",
  "size": "16:9",
  "duration": 5,
  "seed": 42,
  "generate_audio": true
}

场景 2：图生视频（首帧）

{
  "model": "doubao-seedance-2.0",
  "prompt": "小猫站起来走向镜头",
  "image_urls": ["https://example.com/cat.jpg"],
  "duration": 5
}

场景 3：首尾帧视频

{
  "model": "doubao-seedance-2.0",
  "prompt": "从白天过渡到夜晚",
  "image_with_roles": [
    { "url": "https://example.com/day.jpg", "role": "first_frame" },
    { "url": "https://example.com/night.jpg", "role": "last_frame" }
  ],
  "duration": 5
}

场景 4：视频参考生视频

{
  "model": "doubao-seedance-2.0",
  "prompt": "将视频风格转换为动漫风格",
  "video_urls": ["https://example.com/reference.mp4"]
}

场景 5：参考视频 + 参考音频

{
  "model": "doubao-seedance-2.0",
  "prompt": "人物说话的场景",
  "video_urls": ["https://example.com/reference.mp4"],
  "audio_urls": ["https://example.com/speech.wav"],
  "size": "16:9",
  "duration": 11
}

场景 6：有声视频

{
  "model": "doubao-seedance-2.0",
  "prompt": "男人叫住女人说：\"你记住，以后不可以用手指指月亮。\"",
  "generate_audio": true
}

场景 7：连续视频生成（返回尾帧）

{
  "model": "doubao-seedance-2.0",
  "prompt": "小猫继续走向镜头",
  "image_urls": ["https://example.com/last_frame_from_prev.png"],
  "return_last_frame": true
}

场景 8：快速版生成

{
  "model": "doubao-seedance-2.0-fast",
  "prompt": "城市夜景延时摄影",
  "size": "21:9",
  "duration": 8
}

场景 9：使用 Asset URL 图生视频

审核通过的虚拟人像素材可直接作为参考图片传入，无需重新上传和审核。

{
  "model": "doubao-seedance-2.0",
  "prompt": "人物在城市街道自然行走，阳光明媚",
  "image_urls": ["asset://asset_a"],
  "duration": 5,
  "resolution": "720p"
}

场景 10：使用 Asset URL 指定参考人像

{
  "model": "doubao-seedance-2.0",
  "prompt": "使用参考人像，人物优雅地走向镜头",
  "image_with_roles": [
    { "url": "asset://asset_a", "role": "reference_image" }
  ],
  "resolution": "720p",
  "duration": 5
}

查询任务结果

视频生成为异步任务，提交后返回 task_id。使用获取任务状态接口查询生成进度和结果：

GET /v1/tasks/{task_id}
