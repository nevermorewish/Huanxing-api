Codex

codex 第一次启动就报 401

多半是 ~/.codex/auth.json 没创建或内容格式错。它必须是合法 JSON，且包含 OPENAI_API_KEY 字段，例如：{"OPENAI_API_KEY":"sk-..."}。

提示 Responses API 不可用 / 路径 404

确认 base_url 末尾带 /v1；确认 wire_api = "responses"；确认寰星云科该渠道开启了 Responses API 支持。可用 curl 验证：curl https://api.huanxing.ai/v1/responses -H "Authorization: Bearer sk-xxx" -d '{"model":"gpt-5.4","input":"hi"}'。

报错 invalid character '<' looking for beginning of value

Base URL 漏写 /v1，或上游返回了 HTML 错误页。检查 config.toml 中的 base_url。

提示 unknown model

模型名拼写错，或模型未在 default 分组开放。到模型广场把字符串原样复制过去，并确认所在分组包含 default。

config.toml 改了不生效

Codex 默认会缓存 provider 配置，关掉当前 codex 进程，新开终端再启动。如果还不生效，检查路径是不是 ~/.codex/config.toml（不是 ~/.codex/config）。

web_search_request 开了但没联网检索

需要当前模型与上游渠道都支持联网检索。Codex 在不支持的模型上会静默忽略该选项。可换成支持 web search 的 GPT-5 / GPT-5.4 系列再测。

disable_response_storage 应该开还是关

自托管 / 中转场景下建议 true，避免上游帮你存请求；如果你依赖 OpenAI 的 Responses 存储能力做后续分析，可设 false。
